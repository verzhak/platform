Подробное описание

Интерфейс сторонних модулей - реализаций алгоритмов обработки изображений

Заголовочный файл совместим со стандартом C++11

В пользовательских модулях необходимо реализовать следующие функции:

  • process_init();
  • process_destroy();
  • process_memory();
  • process_opencv().

int process_init ( int *  is_opencv_interface)

Инициализация модуля

Аргументы
is_opencv_interface- модуль возвращает в данной переменной 0, если им не поддерживается интерфейс к OpenCV (не реализована функция process_opencv()), и <> 0 в противном случае.
Возвращает
0 - в случае успешного выполнения инициализации;
<> 0 - в случае, если инициализировать модуль не удалось.

Обработка кадра

Аргументы
src- указатель на буфер, содержащий исходное изображение;
dst- указатель на буфер, в котором будет сохранено результирующее изображение;
height- количество строк в обоих изображениях;
width- количество столбцов в обоих изображениях;
channels- количество каналов в src.

Изображения хранятся в памяти развернутыми построчно. Таким образом, доступ ко k-му каналу пикселя, находящегося в i-ой строке и j-том столбце осуществляется с помощью инструкции: src[(i * width + j) * channels + k].

Изображение dst интерпретируется далее как бинарное (!). 0 - рассматривается как 0, <> 0 - как 1.

Возвращает
0 - в случае успешной обработке кадра;
<> 0 - в случае, если обработать кадр не удалось.
int process_opencv ( const void *  src,
void *  dst 
)

Обработка кадра (OpenCV-интерфейс)

Аргументы
src- указатель на исходное изображение.
dst- указатель на результирующее изображение.

Указатели src и dst свободно приводятся к указателю на Mat.

Изображение dst интерпретируется далее как бинарное (!), в котором 0 - рассматривается как 0, <> 0 - как 1.

Возвращает
0 - в случае успешной обработке кадра;
<> 0 - в случае, если обработать кадр не удалось.
